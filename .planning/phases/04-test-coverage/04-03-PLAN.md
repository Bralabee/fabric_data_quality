**Test cases (6 branches):**

1. `test_no_thresholds_all_pass_is_success`:
   Config: no quality_thresholds, no threshold arg. DataFrame: all expectations pass.
   CHARACTERIZATION: Result should have success=True.

2. `test_no_thresholds_any_fail_is_failure`:
   Config: no quality_thresholds, no threshold arg. DataFrame: at least one expectation fails.
   CHARACTERIZATION: Fallback enforces 100% success. Result should have success=False and threshold_failures should mention "100%".

3. `test_global_threshold_at_100_uses_gx_success`:
   Config: no quality_thresholds. Call validate with threshold=100.

   CHARACTERIZATION: threshold >= DEFAULT_VALIDATION_THRESHOLD delegates to GX validation_result.success. Result should have success=False.

4. `test_global_threshold_below_100_compares_rate`:
   Config: no quality_thresholds. Call validate with threshold=50.
   DataFrame: 2 expectations, 1 passes (50% rate). Use expectations where exactly one of two will pass.
   CHARACTERIZATION: 50% rate == 50% threshold. Check whether this is a pass or fail (capture actual behavior).

5. `test_severity_thresholds_checked_independently`:
   Config: quality_thresholds with critical=100, low=0.
   Expectations: one critical (will pass), one low (will fail).
   DataFrame: critical expectation passes, low expectation fails.
   CHARACTERIZATION: Critical at 100% passes its threshold, low at 0% passes its 0% threshold. Overall should be success=True.

6. `test_severity_threshold_failure_causes_overall_failure`:
   Config: quality_thresholds with critical=100.
   Expectations: one critical expectation that will fail.

For each test, use simple GX expectations that produce predictable results:

Mark each test with a docstring starting with "CHARACTERIZATION:" explaining what behavior is being documented.

APPROACH: For each test, FIRST run the validator to observe actual behavior (use debug prints if needed), THEN encode assertions matching actual output. Do NOT guess what the code does -- let the code tell you.
