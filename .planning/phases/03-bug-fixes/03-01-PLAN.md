  <action>
    **BUG-01 fix in _validate_spark_chunked (lines ~242-285):**

    Replace monotonically_increasing_id with row_number():
    ```python
    from pyspark.sql import Window
    from pyspark.sql.functions import row_number, lit

    window = Window.orderBy(lit(1))
    spark_df_with_id = spark_df.withColumn("__chunk_row_num__", row_number().over(window))
    ```

    Fix chunk filtering to use 1-based consecutive numbering:
    ```python
    lower = chunk_idx * chunk_size + 1
    upper = (chunk_idx + 1) * chunk_size
    chunk_df = spark_df_with_id.filter(
        (spark_df_with_id["__chunk_row_num__"] >= lower)
        & (spark_df_with_id["__chunk_row_num__"] <= upper)
    ).drop("__chunk_row_num__")
    ```

    Remove the `offset` variable and its `offset += chunk_size` increment -- no longer needed since boundaries are computed from chunk_idx directly.

    Add a comment explaining: "row_number() produces consecutive 1-based integers. orderBy(lit(1)) preserves existing row order. Non-deterministic across runs is acceptable -- chunking is a memory optimization, not a row-assignment guarantee."

    **BUG-02 fix in _aggregate_chunk_results (lines ~287-325):**

    Replace the entire method body with the per-expectation averaging approach:
    1. Separate valid_results (no "error" key) from error_results
    2. If no valid results, return error dict with chunk_errors
    3. Compute avg_success_rate = mean of success_rate values across valid chunks
    4. Set evaluated_checks from valid_results[0] (all chunks run same suite)
    5. Get threshold from valid_results[0] (falls back to DEFAULT_VALIDATION_THRESHOLD from constants)
    6. Compute failed_checks as evaluated_checks minus the average successful count (or derive from avg rate)
    7. Keep failed_checks and evaluated_checks in the top-level result for handle_failure compatibility
    8. Deduplicate failed_expectations across chunks (union by expectation type + column, capped at MAX_FAILURE_DISPLAY)
    9. Set success = avg_success_rate >= threshold
    10. Add 'chunks' key with per-chunk breakdown list: each entry has chunk_index, success, success_rate, evaluated_checks, failed_checks, failed_expectations
    11. Add 'chunk_errors' key (None if no errors, list otherwise)

    Ensure the result dict retains ALL keys that handle_failure (line ~438) and _save_results_to_lakehouse read: success, batch_name, suite_name, timestamp, evaluated_checks, successful_checks (can compute from avg), failed_checks, success_rate, failed_expectations, chunked_processing, num_chunks.

    Import DEFAULT_VALIDATION_THRESHOLD from constants if not already imported.
