    Create dq_framework/storage.py following the research patterns exactly:

    1. **ResultStore ABC**: Four abstract methods (write, read, list, delete) with type hints as specified in 05-RESEARCH.md Pattern 1. Use `dict[str, Any]` for data, `Optional[str]` for prefix in list().

    2. **JSONFileStore(ResultStore)**:
       - __init__(results_dir="dq_results"): store Path, mkdir(parents=True, exist_ok=True)
       - write(key, data): filter out "validation_result", json.dumps with indent=2 and default=str, write to {key}.json
       - read(key): check exists, json.loads, raise FileNotFoundError if missing
       - list(prefix): glob("*.json"), extract stems, filter by prefix if given, return sorted
       - delete(key): unlink if exists return True, else return False

    3. **LakehouseStore(ResultStore)**:
       - __init__(results_dir="Files/dq_results"): lazy import get_mssparkutils and FABRIC_AVAILABLE from utils. Raise RuntimeError if not FABRIC_AVAILABLE or mssparkutils is None.
       - write(key, data): filter validation_result, json.dumps, mssparkutils.fs.put with overwrite=True
       - read(key): mssparkutils.fs.head(path, 10_000_000), json.loads
       - list(prefix): mssparkutils.fs.ls, filter .json and not isDir, extract stems, filter prefix, sort
       - delete(key): try mssparkutils.fs.rm(path, recurse=False) return True, except return False

    4. **get_store(results_dir=None, backend=None) -> ResultStore**:
       - Import _is_fabric_runtime from utils (inside function to avoid circular imports)
       - If backend="local" or (backend is None and not _is_fabric_runtime()): return JSONFileStore
       - Else: return LakehouseStore
       - Default results_dir: "dq_results" for local, "Files/dq_results" for Fabric

    Use logging.getLogger(__name__) for all log calls. Follow existing codebase patterns.
